    # 图片规范

    # 常用图片格式

    png gif jpeg webp

    # 使用场景

    1. png

    场景：需要透明度，图片格式较小。

    原因：png支持透明通道，但是图片体积偏大，不适合大图

    注意：png又分png-8和png-24（前者体积偏小，支持半透明，后者全透明）

    1. gif

    场景：网页背景、小图标、色彩度低的小切片、动画图片, loading。

    注意：只支持web色颜色种类有限，无法过渡平滑

    1. jpg(jpeg)

    场景：背景图，大图

    原因：色彩还原度比较好，可以支持适当压缩后保持比较好的色彩度

    注意：jpg不支持直接打印，jpeg支持

    4.webp

    google推出的图片格式，真实性好，体积小，但是目前浏览器支持度不高

    5.CSS Sprites(精灵图)

    场景：小的icon图标合并精灵图，减少http请求

    注意：

    1. 适合使用频率高更新频率低的小图标
    2. 尽量不留太多的空白
    3. 体积较大的图片不合并
    1. Data URIs（base64编码）

    场景：

    1. 适合更新频率高的小图片，如某些具备自定义功能的标题icon等
    2. 转换成 Base64 编码的图片应小于 2KB
    3. 移动端不使用 Base64 编码
    4. 要兼容 IE6/IE7 的不使用

    注意：

    1. 转成base64,意味着图片转化为css样式表，有样式改动就会重新下载，不利于缓存（图片可以做缓存，或者cdn）
    2. base64压缩效果不明显,编码的图片尽可能小
    3. 图片不会阻塞渲染，但是样式会阻塞渲染（不是阻塞请求）
